# arist-begin

## 概述
本技能的通用开发规则（问候、TDD、编码门禁、Bug 修复等）已通过 `arist-init.sh` 写入各大模型的全局配置文件，无需在每次对话中重复加载。

本技能启用时仅负责：
1. 确认全局规则已生效
2. 提供联动指引

## 触发条件
- 当用户输入包含 `arist-begin` 时，立即启用本技能。

## 启动检查
启用时执行以下检查：
1. 确认全局配置文件存在且包含开发规则
2. 若发现全局配置缺失或过旧，提示用户运行 `arist-init.sh` 重新配置

## 联动规则（与 arist-end）
- 当用户输入「已完成」时，优先提示并切换使用 `arist-end` 执行标准收尾流程。
- 若当前目录不是 git 仓库，先明确告知无法执行提交/推送，再按用户要求继续非 git 收尾动作。
- 若用户明确要求"不要提交或推送"，联动时仅执行清理与验证步骤，不执行 `git commit` 和 `git push`。

## 全局规则已覆盖的内容（无需在此重复）
以下规则已写入各大模型全局配置文件（`GEMINI.md` / `CLAUDE.md` / `AGENTS.md`）：
- 问候规则（「大哥！」）
- 代码注释（中文）
- 变量命名（camelCase）
- 修改安全性
- 编码前强制门禁
- TDD 强制流程
- Bug 修复规则
- 完成任务规则
- 输出与汇报要求
- 纠错追加规则

## 纠错自学习规则
- 每次用户纠正后，必须在 `templates/global-rules.md` 追加对应的纠错规则，然后提示用户重新运行 `arist-init.sh` 同步到所有模型。
